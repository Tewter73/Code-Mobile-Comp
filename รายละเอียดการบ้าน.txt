📝 การบ้านส่วนที่ 1: HTML + CSS (การจัดหน้า Page Layout)
โจทย์คือให้ออกแบบและจัดหน้าเว็บในหัวข้อที่สนใจ (เช่น สัตว์เลี้ยง, แฟชั่น, กีฬา, หรือเมืองท่องเที่ยว เช่น London, Paris, Tokyo) โดยมีข้อบังคับดังนี้
:
โครงสร้างหน้าเว็บ: ต้องแบ่งสัดส่วนหน้าจอให้ชัดเจน (เช่น มี Header, เมนูด้านซ้าย, เนื้อหาตรงกลาง และ Footer)
.
การจัดการเนื้อหา:
ต้องใส่ "ภาษาไทย" เข้าไปในเนื้อหาด้วย เพื่อทดสอบการแสดงผลของฟอนต์
.
เมื่อคลิกเมนู (เช่น London, Paris, Tokyo) เนื้อหาตรงกลางจะต้องเปลี่ยนตามไป (อนุญาตให้ใช้ JavaScript หรือวิธีอื่นๆ ช่วยได้)
.
กฎการใช้ CSS (สำคัญมาก):
บังคับใช้ External CSS
.
ต้องทำไฟล์ CSS มา 2 ไฟล์ คือ ไฟล์สำหรับแสดงผลแบบ "ขาว-ดำ" (Black and White) และ ไฟล์สำหรับแสดงผลแบบ "สี" (Color) เพื่อให้สามารถสลับเวอร์ชันสีของเว็บได้ตอนตรวจงาน
.
ใช้ฟอนต์ "Sarabun" จาก Google Fonts โดยกำหนดให้ฟอนต์เป็นสีดำและไม่มีการขีดเส้นใต้
.
พื้นหลัง (Background) ต้อง "ชิดขอบ Browser โดยไม่มีช่องว่าง" (Margin เป็น 0)
.
จุดดักคะแนน (Footer Sticky):
ส่วน Footer ด้านล่างสุดที่ใส่ "รหัสนิสิตและชื่อ" จะต้อง "อยู่ติดขอบจอด้านล่างเสมอ" แม้ว่าเนื้อหาตรงกลางจะมีน้อย หรือผู้ใช้จะย่อ/ขยายความสูงของ Browser ก็ตาม (คล้ายเป็นยางยืด)
.
คำใบ้จากอาจารย์: ให้ใช้ CSS Flexbox (display: flex; flex-flow: column; height: 100%;) เข้ามาช่วยจัดการ
.

📝 การบ้านส่วนที่ 2: JavaScript (แบ่งเป็น 3 ข้อย่อย)
ข้อ 2.1 โปรแกรมคำนวณภาษีเงินได้บุคคลธรรมดา
สร้างฟอร์มสำหรับกรอกรายได้ โดยมีช่อง "รายได้", "อัตราภาษี", "ภาษีที่ต้องจ่าย" และ "รายได้รวม"
.
ช่องภาษีที่ต้องจ่ายและรายได้รวม ต้องตั้งเป็น Read-only และให้ระบบคำนวณเปลี่ยนค่าให้อัตโนมัติทันทีที่กรอกรายได้
.
เขียนโค้ดแจ้งเตือน (Alert) หากผู้ใช้กรอกตัวเลขติดลบ หรือใส่ค่าที่ไม่ใช่ตัวเลข (non-numeric)
.
ต้องมีปุ่ม + และ - เพื่อกดเพิ่ม/ลดช่องกรอกรายได้ (จำลองกรณีมีรายได้หลายทาง) โดยบังคับว่า เพิ่มได้รวมไม่เกิน 3 ช่อง และห้ามลบช่องแรกทิ้ง
.
ข้อ 2.2 ทำปุ่มสลับโหมดหน้าจอ (Toggle Dark/Light Mode)
สร้างปุ่มกด เพื่อให้สลับโหมดหน้าเว็บเป็น โหมดมืด (Dark Mode) และ โหมดสว่าง (Light Mode) ไปมาได้ โดยใช้ JavaScript เข้าไปสั่งแก้ไข Style บนหน้าเว็บ
.
ข้อ 2.3 การดึงข้อมูลวันเวลาด้วย Fetch API
ให้ใช้คำสั่ง fetch() ดึงข้อมูลจาก API ของภาครัฐคือ: https://learningportal.ocsc.go.th/learningspaceapi/localdatetime
.
ข้อบังคับการใช้คำสั่ง: บังคับให้ใช้ native fetch() ร่วมกับ .then(), .catch(), และ .finally() (ไม่อนุญาตให้ใช้ Library ภายนอกอย่าง Axios ในงานนี้)
.
การแสดงผล: เมื่อดึงข้อมูล JSON มาได้ ให้นำมาจัด Format แสดงบนหน้าจอเป็น ภาษาไทย เช่น "วันที่ 10 มกราคม 2568 เวลา 10:00:00 น." และเขียนโค้ดให้อัปเดตเวลาทุกๆ 1 วินาที
.
การดักจับ Error: ต้องมีการเขียนโค้ดเพื่อจัดการกรณีที่เกิด Network Error หรือ Connection Timeout
วิธีจำลอง Error: ให้ตั้งค่า Timeout สั้นๆ (เช่น 3 วินาที) หรือใช้คำสั่ง throw new Error('network connection timeout') ดื้อๆ ไปเลย หรือลองปิดเน็ตดู
.
เมื่อเกิด Error โค้ดจะต้องวิ่งไปเข้าบล็อก .catch() และแสดงข้อความเตือนบนหน้าจอว่า "ระบบเครือข่ายล้มเหลว"
